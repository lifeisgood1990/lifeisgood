3.建立联合索引
 如何选择联合索引顺序
     1.经常会被使用到的列优先
     2.选择性高的列优先（离散度大的列 即 select count（distinct column_name） from table 值越大），但实际情况往往会不一样
     3.宽度小的列优先
联合索引匹配规则：最左匹配 or联合索引无效
idx_a_b_c(a,b,c) 能够可以支持a | a,b| a,b,c 3种组合进行查找，但不支持 b,c进行查找
向表中添加一条联合索引

表中重新创建索引（符合创建联合索引：选择性高的列优先）
在webchat_lbs 中 people_id的总数最多，所以people_id列应放在联合索引最左边

通过执行explain sql执行计划，发现没有用到刚刚创建的索引idx_p_m_l,于是通过使用force index 或 use index强制使用某一个索引


通过sql的执行计划，可以看出查询索引用的是刚刚新建的idx_p_m_l，extra用的是覆盖索引而且没有创建临时表和排序，但rows显示扫描了5244932行记录，执行这条sql语句发现耗时2.265s，比使用idx_l_m_p索引多消耗1.7秒多。
4.覆盖索引（use index）
即数据查询可以通过索引查询，不需要通过回表查询
索引的条数小于表的记录数，只读索引，减少I/O操作，尽量使用覆盖索引，可以通过创建联合索引
