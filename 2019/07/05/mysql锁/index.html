mysql锁

1.悲观锁

操作数据时默认认为其他人也会对数据进行修改，分为排他锁和共享锁。

排他锁（独占锁，写锁）

获取锁后，其他事务不能对该数据进行读写，select操作加排他锁select * from T ··· for update，update、delete、insert操作默认会加排他锁。

终端A，开启事务A

终端B，开启事务B，事务B想要对name为tom这行数据操作，必须等待事务A提交

注意:

1.必须开启事务，否则单独的使用是不会加锁的;

2.name字段加索引是行级锁，对同一行进行查询会产生排他锁，如果对其他行查询则不会产生排他锁;

3.如果name字段没有索引，则对name查询会产生表级锁，对不同列行查询也会产生排他锁。

共享锁（读锁）

任何事务只能对数据进行读取，不能修改数据直到其他事务提交。

终端A，开启事务A

终端B，开启事务B

2.乐观锁

操作数据时默认其他用户不会对数据进行操作

通过对表添加一个version字段，每次查询获取version的值，更新是将version作为where条件，或者根据你查询出的字段值进行判断，比如库存stock。
