前后端分离跨越问题

发表于 2017-12-30

最近项目中要采用前后端分离方式开发，采用json格式数据交互。通常解决跨域问题可以使用Jsonp 和 CORS 方式，但Jsonp只支持get请求。所以在项目中采用了CORS,前端ajax请求配置如下:

crossDomain: true，这里代表使用跨域请求
xhrFields: {withCredentials: true}，这样配置就可以把 cookie 带过去了，不然我们连 session 都没法维护,每次访问session都不同
需要注意是，当配置了withCredentials = true时，必须在后端增加 response 头信息Access-Control-Allow-Origin，且必须指定域名，而不能指定为*。当前示例所采用的Java端配置代码：
